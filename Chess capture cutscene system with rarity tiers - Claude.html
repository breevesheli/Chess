The game currently has a screen flashing/flickering issue that repeats continuously and is very disruptive. Diagnose and fix all of the following known causes of canvas flickering:

1. Double-buffering / clearRect on every frame If the main game loop is calling ctx.clearRect() or ctx.fillRect() on every requestAnimationFrame tick even when nothing has changed, it will cause a visible flicker. Fix this by implementing a dirty flag system — only redraw the canvas when needsRedraw = true. Set the flag to true only when: a piece moves, a piece is selected/deselected, a highlight changes, a clock updates, or a cutscene fires. Do not redraw on every tick unconditionally.

2. Canvas being resized on every frame If canvas.width or canvas.height is being set anywhere inside the animation loop or inside a function that runs repeatedly, it will wipe and reflash the canvas every time. Canvas dimensions must only be set once on init and again only on window resize. Move any canvas sizing logic outside the render loop entirely.

3. requestAnimationFrame loop running during cutscenes If the main board render loop continues running while a cutscene overlay is active, the board underneath will keep redrawing and flashing through the overlay. Add a cutsceneActive boolean flag. When a cutscene starts, set cutsceneActive = true and pause the main render loop (do not call requestAnimationFrame again until the cutscene's onComplete() fires and sets cutsceneActive = false).

4. CSS transitions or animations on the board or overlay conflicting If any CSS animation, transition, or @keyframes is applied to the canvas element itself, the board wrapper div, or any parent element of the board, it can cause repeated repaints that look like flickering. Audit all CSS and remove any animation/transition from elements that are ancestors of or siblings to the canvas. Animations are only acceptable on overlay elements, toast elements, and badge elements — never on the canvas wrapper or the board container.

5. The rarity intro flash playing repeatedly If the playIntroSequence() function is being called more than once per capture event (e.g. because an event listener is being added multiple times without being removed), the rarity color flash will loop. Fix this by ensuring dispatchCutscene() can only be called once per capture — add a cutsceneQueued guard flag that blocks re-entry until onComplete() clears it. Also ensure all event listeners for capture events are added exactly once (use { once: true } or remove the listener in the handler).

6. Alpha compositing flicker on overlapping canvas layers If multiple canvas elements are stacked with position: absolute and both are being drawn to on the same frame, semi-transparent regions will flicker as the draw order races. Fix by using a single canvas for the board and a separate single canvas for the cutscene overlay — never draw to both on the same requestAnimationFrame tick. 7. Fix by handling clock display updates inside the requestAnimationFrame loop only — remove any setInterval that touches the canvas. let cutsceneActive = false;
let cutsceneQueued = false;
