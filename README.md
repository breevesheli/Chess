# Chess Popup

A Windows desktop chess application with a custom search-based AI opponent,
learning memory, ELO tracking, opening book overlay, puzzle extraction,
headless training mode, and standalone HTML analysis exports.

Everything runs **entirely locally** — no external services, no API keys.

---

## What's included

**Core app**

| File | Purpose |
| --- | --- |
| `chess_popup_app.py` | PySide6 desktop shell. Launches the local HTTP service and loads the UI in a `QWebEngineView`. |
| `chess_popup_server.py` | Stdlib `http.server` game service. Handles move legality, AI moves, autosave, game records, learning memory updates. |
| `chess_engine.py` | Self-contained chess rules engine + `ChessAI` (minimax with soft-pruning + softmax candidate selection + feature-based learning bias). |
| `chess_popup.html` / `.css` / `.js` | Single-page UI rendered inside the desktop window. |
| `chess_popup.ico` | App / window icon. |

**Tools**

| File | Purpose |
| --- | --- |
| `chess_tools.py` | CLI for opening-book builder, puzzle extractor, headless AI-vs-AI runner, analysis HTML export. |
| `test_chess_engine.py` | Unit tests for engine rules (castling, en passant, promotion, check, draws). |
| `test_chess_popup_server.py` | Integration tests for the HTTP service. |

**Build + install**

| File | Purpose |
| --- | --- |
| `ChessPopup.spec` | PyInstaller spec. |
| `build_chess_popup_exe.ps1` | One-shot rebuild script. |
| `install_chess_popup_shortcut.ps1` | Builds the app and creates a desktop shortcut with the app icon. |
| `run_chess_popup.ps1` / `Run Chess Popup.cmd` | Launches the app from source (bypasses the exe). |

**Persistent data** (local-only; all gitignored)

| File / Folder | What's in it |
| --- | --- |
| `game_records/` | Every finished game saved as `<timestamp>_<mode>_<color>.json` + a sibling `.pgn`. Each JSON has full moves, AI decisions with probabilities, analysis panels, and headers. |
| `bot_learning.json` | Weighted feature biases per mode (`legal` / `illegal`). Updated at end-of-game by `LearningMemory.update()`. |
| `player_stats.json` | Your running ELO (starts 1200), game count, wins/losses/draws. Used by `auto` difficulty. |
| `current_game_autosave.json` | Autosave of the in-progress game. Resume-enabled at next launch. |
| `opening_book.json` | Optional overlay on top of the hardcoded opening book. Built from wins by `chess_tools.py build-opening-book`. |
| `puzzles.json` | Generated by `chess_tools.py extract-puzzles`. Positions where the AI had a clearly dominant best move. |
| `headless_match_log.jsonl` | Append-only log of each AI-vs-AI game run via `chess_tools.py headless`. |
| `analysis_exports/` | HTML analysis files generated by `chess_tools.py export-analysis`. |

## Running it

### Option A — from source (recommended during development)

```powershell
cd "C:\Users\breeves\Desktop\VS Code\Chess Popup"
python chess_popup_app.py
```

or

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\run_chess_popup.ps1
```

Requires Python 3.11+ and `PySide6`. The app launches a local HTTP server
on `127.0.0.1:<random-port>` and opens it in an embedded WebEngine view.

### Option B — the packaged exe

```powershell
.\dist\ChessPopup\ChessPopup.exe
```

The exe is a PyInstaller bundle. **Static assets (HTML / CSS / JS) are
embedded inside `dist\ChessPopup\_internal\` at build time**, so edits to
the source files are ignored until you either:

1. Rebuild the exe: `.\build_chess_popup_exe.ps1`, or
2. Copy the updated files directly into `dist\ChessPopup\_internal\`
   (quick fix for HTML/CSS/JS only — Python changes still need a rebuild).

### Option C — smoke-test the service without the UI

```powershell
python chess_popup_app.py --smoke-test
```

Starts the service, hits `/api/state`, prints "Smoke test passed", exits.

### Data storage note

When running from source, game data writes into the repo folder. When
running the packaged exe, game data writes under `%LOCALAPPDATA%\ChessPopup`
instead (so you can delete/reinstall the exe without losing your record).
Override with the `CHESS_POPUP_DATA_DIR` environment variable if you want a
custom location.

## Features

### Difficulty

Four settings exposed in the UI dropdown:

| Setting | Depth | Temperature | Target |
| --- | --- | --- | --- |
| Easy | 1 | 480 | Beginners / intentional losses |
| Medium | 2 | 220 | Default |
| Hard | 3 | 95 | Serious play |
| **Auto** | Computed | Computed | Matches your current ELO from `player_stats.json` |

Auto difficulty maps rating → profile:

| Rating range | Depth | Temperature |
| --- | --- | --- |
| < 1000 | 1 | 460 |
| 1000–1199 | 2 | 320 |
| 1200–1399 | 2 | 220 |
| 1400–1599 | 2 | 150 |
| 1600–1799 | 3 | 120 |
| 1800+ | 3 | 80 |

Higher temperature = AI picks from a wider candidate set (more mistakes,
more variety). Lower = engine commits harder to its top move.

### Bot modes

- **Legal Bot** — always plays legal moves.
- **Illegal Bot** — may play moves that break movement rules (but never
  king-capture glitches). Four personalities (`standard`, `chaotic`,
  `greedy`, `sneaky`) each tweak the candidate scoring function differently.
  Fun novelty mode; also a useful stress test for the engine.

### Opening book

Two layers, applied in order:

1. **`opening_book.json`** (personal overlay) — generated from your own
   game records. Takes priority if present. Extends to 12 plies.
2. **Hardcoded `OPENING_BOOK`** in `chess_popup_server.py` — small,
   curated. Fallback for positions the overlay doesn't cover.

Only games the AI won or drew are counted when building the overlay.

### Learning memory

`bot_learning.json` stores a weights table keyed by feature strings
extracted per move (piece kind, ply bucket, captures, check bits, etc.).
At end-of-game, `LearningMemory.update()` adjusts the weights by the game
outcome, with later moves weighted slightly higher (position 0 weight 1.0,
position N weight ~1.N). This biases the AI toward move patterns that
worked before. Weights are clamped to ±120 to prevent runaway drift.

### Captured pieces + board layout

Captured pieces render as horizontal strips **above and below the board**,
always on the side of the player who lost them. When you play as Black,
the board flips and the strips automatically swap content.

### Clock behavior

Five time modes: Unlimited, Bullet, Blitz, Rapid, Custom. Clock values
update only when a move happens — **there is no per-second poll/redraw in
the UI**, so selecting a new game setting never gets clobbered by a
background refresh. This was a source of real bugs in earlier versions.

### Analysis

Every finished game's `analysis` section lists best moves, blunders, and
missed tactics. The UI shows these in the side panel under "Analysis".
For a printable export with position diagrams, see
`chess_tools.py export-analysis` below.

### Sounds

Move, capture, check, and game-over chimes use the Windows `winsound`
module (`SystemAsterisk`, `SystemExclamation`, etc.). Toggle with the
Mute button in the toolbar.

## `chess_tools.py` subcommands

### Build an opening book from your games

```powershell
python chess_tools.py build-opening-book [--min-count 2] [--max-ply 12]
```

Scans `game_records/*.json`, aggregates AI moves from wins+draws, writes
`opening_book.json`. The server picks up the overlay on next launch.

### Extract puzzles from your games

```powershell
python chess_tools.py extract-puzzles [--min-probability 0.55] [--min-ply 8]
```

Finds positions where the AI had a dominant best move (probability ≥ 0.55
after ply 8) and writes each position to `puzzles.json` with the expected
UCI move, SAN, and full move-history prefix. Intended as a data source for
a future puzzle-solve UI.

### Run headless AI-vs-AI games for overnight training

```powershell
python chess_tools.py headless --games 50 --mode legal --depth 2 --temperature 220
```

Plays N AI-vs-AI games, updates `bot_learning.json` after each game, and
appends each game's result to `headless_match_log.jsonl`. Useful for
benchmarking engine changes or letting the learning memory converge
overnight.

Options:

- `--no-learning` — play games but don't update the learning memory.
- `--games` — how many to play (default 10).
- `--mode` — `legal` or `illegal`.
- `--depth` / `--temperature` — override the default search parameters.

There's a 200-move safety cap per game.

### Export a game to standalone HTML analysis

```powershell
python chess_tools.py export-analysis "game_records\20260413_101722_legal_white.json"
```

Writes a self-contained HTML file to `analysis_exports/` with:

- Header with result, winner, bot mode, difficulty, outcome
- SVG board diagrams at the start, every 4 plies, and the final position
- Analysis sections (best moves, blunders, missed tactics)
- Full PGN in a preformatted block

Open in any browser and Ctrl+P → Save as PDF for a printable record.

## HTTP API

The local service exposes these endpoints (served on whatever port the app
picked at launch):

| Method | Path | What it does |
| --- | --- | --- |
| GET | `/app.html` | Main UI (served from static files). |
| GET | `/chess_popup.css` / `.js` / `.ico` | Static assets. |
| GET | `/api/state` | Full state payload (board, available moves, clock, AI summary, analysis, saved games, stats). |
| POST | `/api/start` | Start a new game. Body: settings object. |
| POST | `/api/resume` | Resume from `current_game_autosave.json`. |
| POST | `/api/preferences` | Update theme / piece set / auto-flip / muted. |
| POST | `/api/move` | Play a human move. Body: `{from, to, promotion}`. |
| POST | `/api/ai-move` | Ask the bot to move. |
| POST | `/api/undo` | Undo the last pair of moves. |
| POST | `/api/restart` | Restart with the same settings. |
| POST | `/api/resign` | Resign the current game. |
| POST | `/api/draw` | Offer/accept a draw. |
| POST | `/api/hint` | Return the hardcoded-hard-mode best move for the current side. |
| POST | `/api/load-record` | Load a saved game into replay mode. |

All responses are JSON. Errors return 4xx/5xx with `{"error": "..."}`.

## Building the exe

```powershell
.\build_chess_popup_exe.ps1
```

This runs PyInstaller against `ChessPopup.spec`, writes the output to
`dist\ChessPopup\`, and leaves `dist\ChessPopup\ChessPopup.exe` ready to
run. The `build/` folder is a PyInstaller work area and can be deleted
anytime.

**Important:** after editing any of `chess_popup.html`, `.css`, or `.js`,
the exe will not see the changes until either:

- You rebuild, or
- You manually copy the files into `dist\ChessPopup\_internal\`.

After editing any `.py` file, you **must** rebuild — Python code is frozen
into the bundle.

## Troubleshooting

### "The app freezes / my settings reset before I click Play"

You are running a stale packaged exe whose embedded JS predates the
clock-poll fix. Rebuild the exe (`.\build_chess_popup_exe.ps1`) or copy
`chess_popup.js` / `.html` / `.css` into `dist\ChessPopup\_internal\`
manually.

### "Pieces look thin and hard to see"

Both sides now use the filled Unicode glyphs (U+265A–U+265F) with
contrasting outlines. If they still look wrong, your font fallback chain
is probably missing `Segoe UI Symbol`. Check the `.piece` rule in
`chess_popup.css`.

### "The bot autosave keeps resuming a game I don't want"

Delete `current_game_autosave.json`. It's recreated on the next new game.

### "Learning memory drifted and now the bot plays weird"

Delete or rename `bot_learning.json`. The file will be recreated fresh on
the next finished game.

### "Exe won't open"

Check that `dist\ChessPopup\_internal\` exists and contains the full
Python/PySide6 runtime. Rebuild with `.\build_chess_popup_exe.ps1` if
anything was deleted.

### "Port already in use"

The service binds to an ephemeral port by default, so collisions shouldn't
happen. If you force a specific port and it's in use, pass `--port 0` to
let the OS pick.
